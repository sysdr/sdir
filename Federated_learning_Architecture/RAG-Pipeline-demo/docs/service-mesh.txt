A service mesh offloads cross-cutting concerns like mutual TLS, circuit breaking, retries, and distributed tracing from application code into a sidecar proxy running alongside each service. The data plane consists of proxies intercepting all inter-service traffic; the control plane distributes configuration and policy to proxies. Envoy is the dominant data plane proxy, used by both Istio and AWS App Mesh. The primary cost is latency overhead: a request traverses two proxies, adding 1-3ms per hop depending on configuration. CPU overhead is approximately 0.5 cores per 1000 requests per second. Observability is the clearest win: service meshes provide request counts, latency histograms, and error rates for every service pair without code changes. The major operational challenge is managing the control plane at scale across thousands of proxies.
