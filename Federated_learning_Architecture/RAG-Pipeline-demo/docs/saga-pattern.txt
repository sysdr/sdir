The Saga pattern manages long-running distributed transactions by decomposing them into a sequence of local transactions, each with a corresponding compensating transaction. If any step fails, the saga executes compensating transactions in reverse order to undo completed steps. There are two coordination approaches: choreography, where each service publishes events that trigger the next step, and orchestration, where a central coordinator directs the sequence. Choreography is decentralized and resilient but hard to debug when failures occur across multiple services. Orchestration makes the flow explicit and visible but introduces a single coordinator that must be highly available. Uber's payment system uses sagas to coordinate charging, trip completion, and driver payout across independent services.
