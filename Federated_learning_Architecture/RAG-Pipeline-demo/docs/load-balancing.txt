Load balancers distribute incoming traffic across a pool of backend servers to maximize throughput and minimize response time. Round-robin assigns requests to servers sequentially regardless of server state. Weighted round-robin accounts for heterogeneous server capacity. Least-connections routes new requests to the server with the fewest active connections, better matching backend capacity for variable-duration requests. The power-of-two choices algorithm samples two servers randomly and routes to the less loaded, achieving near-optimal load distribution with minimal coordination overhead at high scale. Layer 4 load balancers operate on TCP connections and are faster but cannot route based on HTTP content. Layer 7 load balancers parse HTTP and enable content-based routing, A/B traffic splitting, and request rewriting at the cost of higher CPU overhead. NGINX and HAProxy are common software load balancers; AWS ALB/NLB are managed alternatives.
