Rate limiting controls request throughput to protect services from overload. The token bucket algorithm permits burst traffic up to the bucket capacity while enforcing an average rate equal to the token refill rate. The leaky bucket enforces a smooth output rate regardless of burstiness. Fixed window counting resets counters every interval, creating edge-case bursts at window boundaries. Sliding window algorithms eliminate the boundary problem but require more storage. At distributed scale, rate limits are enforced in a shared store like Redis using atomic increment operations with expiry. Redis Lua scripts ensure atomicity of check-and-increment operations. Cloudflare enforces rate limits at the edge before traffic reaches origin servers. Stripe's API uses token buckets with separate limits per endpoint, allowing burst capacity for checkout flows while protecting lower-priority endpoints.
