Cache invalidation and population strategies determine whether cached data stays fresh and what penalty is paid on cache misses. Cache-aside loads data into the cache on demand: on a miss, the application fetches from the database and populates the cache. Write-through writes to the cache and database synchronously on every write. Write-behind buffers writes in the cache and flushes to the database asynchronously, improving write throughput at the cost of potential data loss. Read-through is identical to cache-aside but managed by the cache layer rather than application code. The thundering herd problem occurs when a popular cache entry expires and many concurrent requests all experience a cache miss simultaneously, flooding the database. Probabilistic early expiration refreshes cache entries before they expire using a random probability that increases as the entry ages. Redis supports all these patterns through expiry, pub/sub invalidation, and Lua scripting.
