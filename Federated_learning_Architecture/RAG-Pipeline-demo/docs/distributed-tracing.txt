Distributed tracing tracks request execution across multiple services by propagating a trace context through all intermediate calls. Each service creates a span representing its work, attaching timing, metadata, and the parent span identifier, then reports it to a collection backend. The trace is the complete tree of spans for a single request. The W3C Trace Context specification standardizes trace propagation headers, enabling interoperability across different tracing systems. Sampling is required at high request volumes: storing every trace is prohibitively expensive, so most systems use head-based sampling at the entry point or tail-based sampling that buffers traces and samples based on error or latency signals. Jaeger and Zipkin are open-source backends; AWS X-Ray and Datadog APM are managed alternatives. The primary debugging value is identifying which service in a long call chain contributed unexpected latency.
