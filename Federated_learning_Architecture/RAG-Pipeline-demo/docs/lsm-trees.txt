Log-structured merge trees optimize for write throughput by converting random writes into sequential writes. All incoming writes are buffered in an in-memory sorted structure called a memtable; when it fills, it flushes to disk as an immutable sorted file called an SSTable. Background compaction merges SSTables to reclaim space from deleted or overwritten values and maintain a bounded number of sorted levels. Read performance degrades with more levels because every read must potentially check multiple SSTables. RocksDB's leveled compaction strategy reduces the number of SSTables per level at the cost of write amplification. Cassandra, HBase, and InfluxDB all use LSM trees as their storage engine. The trade-off versus B-trees is write amplification for write throughput and read amplification.
