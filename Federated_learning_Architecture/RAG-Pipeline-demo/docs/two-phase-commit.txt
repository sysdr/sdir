Two-phase commit coordinates atomic transactions across multiple databases or services. In the prepare phase, the coordinator asks all participants if they can commit. Each participant writes the transaction to a write-ahead log and responds yes or no. In the commit phase, if all participants voted yes, the coordinator sends commit to all; if any voted no, it sends rollback. The critical failure mode is coordinator crash after the prepare phase but before commit. Participants holding locks are blocked until the coordinator recovers, creating availability problems. Three-phase commit adds an additional phase to handle coordinator crashes but adds latency and does not fully solve the blocking problem.
