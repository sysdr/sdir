Gossip protocols propagate information through a cluster by having each node periodically exchange state with a random subset of peers. Information spreads exponentially: if each node infects two others per round, the entire cluster learns new information in O(log n) rounds. This decentralized approach has no single point of failure and tolerates node failures naturally. Cassandra uses gossip for cluster membership, failure detection, and schema distribution. Each node maintains a heartbeat counter that increases monotonically; if a node's heartbeat stops incrementing, it is marked suspect and eventually failed. The convergence time is bounded by the gossip fanout and the network diameter. The downside is eventual consistency: there is a window where nodes have different views of cluster membership, which requires application-level handling of slightly stale routing information.
