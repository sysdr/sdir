Database replication copies data from a primary to one or more replicas for read scaling, disaster recovery, or geographic distribution. Synchronous replication guarantees that a write is not acknowledged to the client until at least one replica has applied it, eliminating data loss at the cost of write latency. Asynchronous replication acknowledges writes immediately on the primary and propagates to replicas out of band, adding throughput but accepting potential data loss if the primary crashes before replication completes. Logical replication ships row-level changes decoded from the WAL, enabling replication between different major database versions or to different database systems. Multi-master replication allows writes on multiple nodes, requiring conflict detection and resolution for concurrent writes to the same row. MySQL Group Replication and PostgreSQL logical replication are widely used implementations.
