Backpressure is a flow control mechanism that propagates load signals upstream to prevent buffer overflow and out-of-memory failures. In reactive systems, a consumer that cannot keep up signals to its producer to reduce the emission rate rather than buffering indefinitely. TCP implements backpressure through the receive window: the receiver advertises available buffer space, and the sender limits its outstanding data accordingly. In message queues, backpressure manifests as producer throttling when queue depth exceeds a threshold. Akka Streams and Project Reactor implement backpressure through the Reactive Streams protocol, where consumers request a specific number of elements from producers. The failure mode for systems without backpressure is heap exhaustion: unbounded queues accumulate until the process runs out of memory and crashes. Kafka producers handle backpressure by blocking or dropping messages based on configuration when the internal send buffer fills.
