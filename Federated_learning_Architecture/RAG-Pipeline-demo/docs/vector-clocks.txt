Vector clocks are data structures that track causality in distributed systems. Each process maintains a vector of logical timestamps, one entry per process. When a process sends a message, it increments its own entry and includes the full vector. The receiver updates its vector by taking the element-wise maximum and incrementing its own entry. If event A's vector is strictly less than event B's vector in every entry, A causally precedes B. If neither dominates the other, the events are concurrent and a conflict resolution policy is required. Amazon DynamoDB used vector clocks for version tracking until the complexity of managing divergent versions led them to adopt last-write-wins with vector clocks for conflict detection only.
