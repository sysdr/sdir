Bloom filters are probabilistic data structures that test membership with zero false negatives and a tunable false positive rate. They use multiple hash functions to set bits in a bit array on insertion; to test membership, they check whether all corresponding bits are set. Adding elements never produces false negatives, but the false positive rate increases as the filter fills. The false positive probability is approximately (1 - e^(-kn/m))^k where k is the number of hash functions, n is the number of inserted elements, and m is the bit array size. Cassandra uses bloom filters to avoid disk reads for keys that definitely do not exist. RocksDB uses bloom filters per SSTable to skip irrelevant files during point lookups. They cannot support deletion without counting bloom filters.
