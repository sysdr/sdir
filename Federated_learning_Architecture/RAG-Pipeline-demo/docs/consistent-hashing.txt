Consistent hashing maps both keys and servers onto a circular ring using a hash function. When a server is added or removed, only k/n keys need to be remapped on average, where k is the number of keys and n is the number of servers. Without consistent hashing, every server change would require remapping nearly all keys. Virtual nodes address the uneven distribution problem by assigning each physical server multiple positions on the ring, improving load balance. Amazon DynamoDB and Apache Cassandra both use consistent hashing with virtual nodes for data partitioning. The primary failure mode is hot spots when virtual node counts are too low, concentrating load on a small number of physical servers.
