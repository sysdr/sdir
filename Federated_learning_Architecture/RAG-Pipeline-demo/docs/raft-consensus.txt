Raft is a distributed consensus algorithm that elects a single leader responsible for managing log replication. All client requests go through the leader, which appends entries to its log and replicates them to followers. An entry is committed once a majority of nodes acknowledge it. Leader election uses randomized timeouts to prevent split votes: each follower waits a random interval before promoting itself to candidate, reducing the likelihood that multiple nodes call elections simultaneously. Log compaction through snapshotting prevents unbounded log growth. Raft deliberately sacrifices some performance for understandability compared to Paxos.
